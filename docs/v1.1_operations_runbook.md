## Daily Operations

### Morning Data Health Check

```bash
# 1. Universe health
python main.py universe-health

# 2. Freshness check
python main.py freshness-health
python main.py freshness-stale

# 3. Quality gate summary
python main.py quality-gate-health
python main.py quality-gate-summary

# 4. Governance dashboard
python main.py governance-health
python main.py governance-summary

# 5. Governance alerts
python main.py governance-alerts-health
python main.py governance-alerts

# 6. Governance rollup (v1.1.9)
python main.py governance-rollup-health
python main.py governance-rollup-consistency
```

### Interpreting Results

- **PASS**: Normal operation
- **WARN**: Monitor; may indicate degraded data or configuration drift
- **FAIL**: Requires investigation; do not proceed with formal research

### Alert Triage

P0 alerts (CRITICAL/HIGH):
1. `AUDIT_CHAIN_FAILURE` → run `gate-enforcement-verify`
2. `SOURCE_INTERRUPTION` → provide real data; no auto-download
3. `ROLLUP_STORE_CORRUPTED` → run `governance-rollup-audits`, then preview recovery

P1 alerts (HIGH):
1. `MODULE_HEALTH_FAIL` → run `governance-module-health`
2. `ROLLUP_CONSISTENCY_FAILURE` → run `governance-rollup-consistency`

P2 alerts (MEDIUM):
1. Review in morning digest: `python main.py governance-digest`

## Pre-Upgrade Operations

Before upgrading to a new version:

1. Run stable checklist:
   ```bash
   python main.py research-cockpit-stable --mode real
   python main.py stable-v060-check --mode real
   ```

2. Run regression suite:
   ```bash
   python main.py regression-run --suite quick --mode real
   ```

3. Run safety scan:
   ```bash
   python main.py safety-scan --target all
   ```

4. Verify governance health:
   ```bash
   python main.py governance-rollup-health
   python main.py governance-rollup-consistency
   ```

5. Check version info:
   ```bash
   python main.py version-info
   ```

All checks must pass before committing an upgrade.

## Cross-Machine Sync

When working across two computers:

1. **Before pushing**: verify portable paths
   ```bash
   python main.py governance-rollup-paths
   ```

2. **Commit only source code** — `data/governance_rollup/` is gitignored

3. **On the other machine**: pull and rebuild stores
   ```bash
   python main.py governance-rollup-store-inventory
   python main.py governance-rollup-health
   ```

4. If stores are missing, run the relevant health commands to regenerate:
   ```bash
   python main.py universe-health
   python main.py freshness-health
   python main.py governance-health
   ```

## Safety Reminders

- `--allow-write` is NEVER required for read commands
- Recovery and migration execution ALWAYS require `--allow-write`
- No store is modified without explicit user action
- All health checks and audits are read-only
- `NO_REAL_ORDERS = True` — no trading actions are ever taken

## Escalation

If governance health is RED and cannot be resolved:

1. Do not proceed with formal research
2. Run `python main.py governance-rollup-report` for a full audit trail
3. Check `data/governance_rollup/` for raw diagnostic data
4. Review alert history: `python main.py governance-alert-history`
5. Consult `docs/troubleshooting.md` for common issues
